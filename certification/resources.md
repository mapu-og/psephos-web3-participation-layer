Resources
The following resources will help you build your project! If you're building a web3 application you'll either want to pick a solution that handles the complete stack for you, like Scaffold-Alchemy, or you can pick the pieces of the stack yourself. Below you'll find resources for both options!

Finally, if you're focused purely on smart contract development you'll want to skip down to that section directly as well.

Scaffold Alchemy
Looking for everything you need to build a web3 application? Scaffold-Alchemy is a great choice.

npx create-web3-dapp@latest
Scaffold-Alchemy is a fork of the popular starter project scaffold-eth 2. It is everything you need to build web3 applications. You can get started immediately NextJS, TypeScript, Hardhat, AccountKit, and Enhanced APIs 🤩

The beauty of Scaffold Alchemy is that you can have lightning fast iteration between your smart contracts and web application code. Make changes to your contracts and immediately you'll have hooks, components and types that recognize these changes.

Pick your stack
Ready to pick all the components of your stack? We'll leave some resources below but feel free to find the tool that best fits for you!

Smart Contract Development
For Smart Contract development you'll likely want to use Hardhat or Foundry.

Hardhat is going to be the better choice for a JavaScript/TypeScript developer who would like to build tests and scripts in those languages. If you'd prefer to be writing Solidity unit tests, check out Foundry.

Front-End Development
For front-end development there is a ton of choice. Maybe too much 

To build the front-end application itself you'll likely want to use a framework. In all of our examples we use react. You'll likely want to pair this with a build tool like vite or a fully-featured framework like next.js.

Beyond that there's many additional tools you may consider for react, depending on your needs. Here's a great resource for finding the appropriate libraries: https://www.robinwieruch.de/react-libraries/

For interacting with blockchains from the front-end you'll want to use a library like viem (you can learn viem here) or ethers.

If you'd like to embed a wallet in your application, check out the Alchemy Account Kit for a zero friction onboarding experience.
