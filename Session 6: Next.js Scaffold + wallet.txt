Session 6: Next.js Scaffold + wallet
Prompt: 
Scaffold un frontend en `alchemy-final/frontend/` para interactuar con el contrato SurveyPlatform deployado en Base Sepolia.

## Datos del contrato
- Address: `0x5caba1c7e5caB1651FFF75Fc335010b92A75bE9b`
- Network: Base Sepolia (chainId 84532)
- ABI: genéralo del contrato adjunto `SurveyPlatform.sol`

## Stack
- Next.js 14 (App Router)
- TypeScript
- Tailwind CSS
- RainbowKit + wagmi + viem

## Páginas a crear
1. **Layout** — header con botón de conectar wallet (RainbowKit), nombre del proyecto
2. **Home `/`** — lista de encuestas activas (llama `getActiveSurveys()` + `getSurvey()` por cada ID). Muestra: título, recompensa, respuestas restantes, deadline
3. **Crear encuesta `/create`** — formulario: título, IPFS hash, reward por respuesta, max respuestas, deadline. Botón que llama `createSurvey()` con el ETH calculado
4. **Detalle encuesta `/survey/[id]`** — muestra toda la info de la survey. Si el usuario no ha respondido: input para answer hash + botón `submitResponse()`. Si ya respondió y no ha hecho claim: botón `claimReward()`. Si ya hizo claim: mostrar "Reward claimed ✅"

## Config
- Crear `frontend/.env.local.example` con `NEXT_PUBLIC_WALLETCONNECT_PROJECT_ID=`
- El ABI y la dirección del contrato van en `frontend/src/config/contract.ts`
- wagmi config en `frontend/src/config/wagmi.ts`

## Criterio de done
- `npm install` sin errores
- `npm run dev` levanta en localhost
- Se puede conectar wallet con RainbowKit
- Se ve la lista de encuestas (puede estar vacía)

## No hacer
- No crear backend/API routes
- No integrar IPFS (por ahora el hash se pone manual)
- No agregar estilos complejos — Tailwind básico y limpio
- No agregar más páginas o features
